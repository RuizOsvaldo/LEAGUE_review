# Ticket 003: Drizzle Config and Migration Reset

## Description

Update `server/drizzle.config.ts` to use the SQLite dialect, then delete the
existing Postgres migration files and generate a fresh SQLite init migration
from the rewritten schema.

The 9 existing migration files (`0000_*.sql` through `0008_*.sql`) and their
corresponding meta snapshots contain Postgres-specific DDL (`CREATE TYPE`,
`timestamp with time zone`, `uuid_generate_v4()`, etc.) that is not valid
SQLite. They must be removed. A single new `0000_init.sql` is generated by
`drizzle-kit generate` against the rewritten SQLite schema.

## Acceptance Criteria

- [x] `server/drizzle.config.ts` declares `dialect: 'sqlite'`.
- [x] `server/drizzle.config.ts` uses `dbCredentials: { url: process.env.DATABASE_URL! }`.
- [x] `server/drizzle.config.ts` no longer loads `.env` via `dotenv.config()` for
      Docker use (the env var is already set by compose/entrypoint); it may keep
      the dotenv load for local development convenience — implementer's choice.
- [x] All Postgres migration `.sql` files in `server/drizzle/` are deleted.
- [x] All meta snapshot `.json` files in `server/drizzle/meta/` are deleted
      (except `_journal.json` which will be regenerated).
- [x] `cd server && npx drizzle-kit generate` runs successfully and produces a
      single `0000_lethal_wong.sql` in `server/drizzle/`.
- [x] The generated migration contains only SQLite-compatible DDL (no
      `CREATE TYPE`, no `GENERATED ALWAYS`, no `timestamp with time zone`).
- [x] `cd server && npx drizzle-kit migrate` applies the migration to a local
      SQLite file without error (requires `DATABASE_URL` to be set).

## Implementation Plan

### Files to Modify

- `server/drizzle.config.ts` — update dialect and credentials

### Files to Delete

- `server/drizzle/0000_new_wind_dancer.sql` through `0008_volunteer_event_schedule.sql`
  (all 9 SQL files and any additional files in the directory)
- `server/drizzle/meta/0000_snapshot.json` through `0007_snapshot.json`
  (all numbered snapshot files)

### Steps

1. Edit `server/drizzle.config.ts`:
   ```typescript
   export default defineConfig({
     schema: './src/db/schema.ts',
     out: './drizzle',
     dialect: 'sqlite',
     dbCredentials: {
       url: process.env.DATABASE_URL ?? 'file:./data/dev.db',
     },
   });
   ```

2. Delete all existing SQL and snapshot files:
   ```bash
   rm server/drizzle/*.sql
   rm server/drizzle/meta/[0-9]*.json
   ```

3. Generate the fresh SQLite migration:
   ```bash
   cd server
   DATABASE_URL=file:./data/dev.db npx drizzle-kit generate
   ```

4. Verify the output: inspect `server/drizzle/0000_init.sql` to confirm
   it uses SQLite DDL only.

5. Apply the migration to confirm it runs cleanly:
   ```bash
   mkdir -p data
   DATABASE_URL=file:./data/dev.db npx drizzle-kit migrate
   ```

### Testing Plan

- Inspect `0000_init.sql` for Postgres-specific syntax (none expected).
- `drizzle-kit migrate` completes without error.
- SQLite file is created at the expected path.
