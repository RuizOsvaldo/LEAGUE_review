# Ticket 001 Plan: feedback_token schema migration

## Approach

Add a `feedbackToken` UUID column to `monthly_reviews` with a default of
`gen_random_uuid()` and a unique constraint. This is a purely additive
migration — no existing columns change.

## Files to Modify

- `server/src/db/schema.ts` — add `uuid` import, add `feedbackToken` column, add unique constraint

## Files to Create

- `server/drizzle/` — migration file (generated by `npm run db:generate`)
- `tests/db/schema.test.ts` — new db tests for the feedback_token column

## Key Design Decisions

- Use `uuid('feedback_token').notNull().defaultRandom()` — Drizzle's wrapper
  around `gen_random_uuid()`, produces a v4 UUID at the DB level.
- Unique constraint ensures no two reviews share the same token.
- Existing rows will have `gen_random_uuid()` applied automatically by the
  migration default, so no backfill query is needed.

## Testing Plan

- Run `npm run test:db` (existing) — verify no regressions.
- Run `npm run test:server` (existing) — verify no regressions.
- New tests in `tests/db/schema.test.ts`:
  - Verify all `monthly_reviews` rows have a non-null `feedback_token`.
  - Verify inserting two rows with the same `feedback_token` raises a unique violation.
